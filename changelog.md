# Changelog

Все заметные изменения в этом проекте будут задокументированы в этом файле.

Формат основан на [Keep a Changelog](https://keepachangelog.com/en/1.0.0/),
и этот проект придерживается [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.0.1.6] - 2026-03-08 (Bandito: Plugin LED Name/Version Display)

### Added
- **Отображение названия и версии плагина в полях plugin_led_1–5 (сервер):**
  - Поля `plugin_led_1`–`plugin_led_5` в `el_gui_bandito.ui` заполняются строкой вида «{name} v{version}» при загрузке и при любом обновлении слота (назначение, Reload, удаление).
  - Добавлен метод `_enrich_slot_data_from_manifest(slot_index)`: при отсутствии `name` или `version` в данных слота они подставляются из манифеста плагина (`*_manifest.json` → `name`, `min_app_version`), в т.ч. для старых конфигов.
  - В `update_slot_ui` добавлена подстановка name/version из манифеста, если в переданных данных они пустые; защита от вывода `None` (подстановка `"?"`).

### Changed
- **Инициализация UI слотов:** вызов `_init_ui_from_core()` перенесён из `BanMainWindow.__init__` в `main()` — выполняется после `window.ui.setupUi(window)`, чтобы виджеты LED существовали к моменту заполнения (устранён `AttributeError: 'BanMainWindow' object has no attribute 'ui'`).
- В `update_slot_ui` добавлена проверка наличия `self.ui`; при её отсутствии метод завершается без ошибки (защита при вызове по сигналу до инициализации UI).

## [0.0.1.5] - 2026-03-08 (Core: Plugin Core Injection & Log Optimization)

### Added
- **Передача Core в плагины:**
  - В `ElCore._load_plugin_instance` при создании плагина пробуется вызов `plugin_class(plugin_path, core=self)`; при `TypeError` — fallback на `(plugin_path)` и `()`.
  - Плагины могут принимать опциональный аргумент `core` и использовать `core.com.broadcast()` для рассылки. Подробности по плагинам — в [changelog_shortcut.md](changelog_shortcut.md), [changelog_tune.md](changelog_tune.md).

### Changed
- **Tune:** broadcast переведён на `core.com.broadcast`; удалены зависимость от `requests` и fallback на прямой POST.
- **Логи (сокращение вывода):**
  - **Сервер (Uvicorn):** `log_level="warning"` и `access_log=False` — убраны 4 строки при старте и лог каждого POST; одна строка `[Core] Server: 0.0.0.0:8000`.
  - **Подключение клиента:** одна строка `[Core] Client: IP` вместо двух (закомментированы полные варианты с пометкой DEBUG).
  - **ElStateManager:** `[ElStateManager] State loaded: ok` вместо пути к конфигу; при ошибках — краткое сообщение.
  - **Старт приложения:** «Проверка UI файлов» и «Проверка UI файлов плагинов» объединены в одну строку `Проверка UI файлов: ok`; «Сервер запущен» заменён на `Server: Online`; вывод «Auto-activating slot N on startup» закомментирован (DEBUG).
- **Очистка кода:** упрощены комментарии в `el_core.py`; закомментированы лишние `print` в `el_state_manager.py` (client connected/disconnected).
- **Логи кнопок (Core):** при `PLUGIN_BUTTON_PRESS` одна строка `[Core] PLUGIN_BUTTON_PRESS → slot N 'btn_id'` вместо двух (Received command + Delegating).
- **Клиент — логи при старте:**
  - Логи апдейтера (app + plugins): уровень `WARNING` — в консоль только предупреждения и ошибки.
  - Убраны `print`: «Запуск Client (Cliento)...», «Проверка обновлений ядра/плагинов», «Клиент запущен.».
  - `QT_LOGGING_RULES` выставляется в начале `el_cliento.py` (до импортов Qt) для подавления вывода qt.multimedia (ffmpeg).
- **ElSoundManager:** при инициализации выводится лог устройства вывода по умолчанию: число устройств и `default: "описание"` (имя бэкенда Qt не отдаёт).

## [0.0.1.3] - 2026-03-02 (Core: Sound System & Audio Management)
### Added
- **Система звуков (ElSoundManager):**
    - Внедрен универсальный менеджер звуков `el_core/el_sound_manager.py` на базе `QSoundEffect` для минимальной задержки.
    - **Audio Device Detection:** Реализована автоматическая проверка наличия аудиоустройств через `QMediaDevices`. При отсутствии звуковой карты система переходит в "тихий режим" без генерации ошибок.
    - **Sound Resources:** Создана централизованная директория `resources/sounds/` для системных звуков.
- **Интеграция в Core & Client:**
    - **ElCore:** Теперь управляет воспроизведением звуков при переключении плагинов и рассылает команду `PLAY_SOUND` всем клиентам.
    - **ElCliento:** Реализован глобальный `eventFilter` для автоматического воспроизведения звука клика при нажатии на любые `QPushButton` или `QToolButton`.
- **Настройки (BanditoSettings):**
    - В интерфейс настроек сервера добавлен чекбокс `sound_checkB` для глобального включения/выключения звуков в системе. Состояние сохраняется в `el_bandito_config.json`.

### Changed
- **Server Side (Update System):**
    - **`server_main.py`**: Улучшена логика `UPDATE_GET_MANIFEST`. Теперь сервер поддерживает рекурсивное сканирование директорий (директива `is_directory: true`), что позволяет автоматически синхронизировать папки со звуками и иконками.
    - Исправлена ошибка поиска файлов в `el_core/` при запуске сервера из подпапки `el_bandito`.
- **Manifests:**
    - `cliento_manifest.json` обновлен для поддержки загрузки `el_sound_manager.py` и синхронизации папки `resources/sounds`.

## [0.0.1.2] - 2026-03-01 (Core: Plugin Orchestrator & UI Optimization)
### Added
- **Plugin Orchestrator (ElCore):**
    - Внедрена архитектура оркестратора в `el_core/el_core.py` для централизованного управления плагинами.
    - **State Management:** Создан `ElStateManager` для хранения конфигурации слотов, активных плагинов и кэширования инстансов в RAM с сохранением в JSON.
    - **Communication Layer:** Создан `ElComManager` — изолированная обертка над WebSocket-сервером для управления сетевыми событиями и широковещательными рассылками (broadcast).
    - **Plugin Life Cycle:** Реализован механизм "бесшовного" переключения плагинов. Плагины теперь загружаются один раз и сохраняют свое состояние (state) в памяти при переключении вкладок.
- **UI Architecture (Server & Client):**
    - Внедрен `QStackedWidget` для отображения плагинов в `right_frame`. Это устранило мерцание и гарантирует сохранность интерфейса и запущенных процессов (например, музыки) при навигации.

### Changed
- **Refactoring:**
    - `el_bandito.py` полностью переписан на использование `ElCore`. Удалена избыточная логика управления сокетами и ручная загрузка UI.
    - Оптимизирована работа с `sys.path`: пути плагинов теперь добавляются без дублирования.
- **Bug Fixes:**
    - Исправлен парсинг версий из JSON-манифестов (теперь извлекается `min_app_version` вместо чтения всего файла).
    - Исправлен конфликт имен модулей `resources` через динамический импорт UI плагинов.
    - Исправлена ошибка `api_qt_size` и перенесена логика обработки кнопок внутрь классов плагинов.
    - Устранена "петля обратной связи" при переключении слотов между клиентом и сервером.

## [0.0.1.14] - 2026-03-01 (Core & Plugins: Manifest-based Versioning)
### Changed
- **Единый источник правды для версий (Core & Plugins):**
    - **Архитектура:** Полный переход от использования разрозненных файлов версий (`ver_cliento`, `ver_shortcut` и т.д.) к хранению версии внутри JSON-манифестов (`min_app_version`).
    - **`cliento_updater.py`**: Теперь считывает локальную версию напрямую из `cliento_manifest.json`. Манифест сохраняется сразу после получения от сервера, что гарантирует атомарность обновления версии.
    - **`cliento_plugin_updater.py`**: Аналогичный рефакторинг для плагинов. Версия каждого плагина теперь берется из его специфичного манифеста (например, `shortcut_manifest.json`).
    - **Server Side (`server_main.py`)**: Обработчики команд `UPDATE_GET_VER` и `PLUGIN_UPDATE_GET_VER` переписаны для динамического извлечения версий из соответствующих манифестов на стороне сервера.
- **Очистка проекта:**
    - Удалены все устаревшие файлы версий (`ver_cliento`, `ver_shortcut`, `ver_sound`, `ver_example`, `ver_music`) из всех директорий.
    - Актуализированы манифесты всех плагинов: поле `version_file` теперь указывает на сам файл манифеста, а в `files_map` старые файлы версий заменены на манифесты.

## [0.0.4] - 2026-01-23
### Added
- **Система автообновления (Auto-Update System):**
    - Реализован механизм автоматического обновления клиента (`el_cliento`) при запуске.
    - **Server Side:** Добавлены WebSocket-команды `UPDATE_GET_VER`, `UPDATE_GET_MANIFEST`, `UPDATE_DOWNLOAD_FILE` для раздачи файлов.
    - **Client Side:** Создан модуль `el_cliento/cliento_updater.py`, который:
        1. Сверяет версию клиента с серверной.
        2. Запрашивает манифест обновлений (`cliento_manifest.json`).
        3. Скачивает и заменяет файлы при необходимости.
        4. Автоматически перезапускает клиент после обновления.
    - **Manifest:** Введен файл `el_cliento/cliento_manifest.json`, описывающий список файлов и директорий, подлежащих синхронизации.
    - **[NEW] MD5 Verification:** Внедрена проверка целостности файлов.
        - Сервер динамически вычисляет MD5 хеши файлов перед отправкой манифеста.
        - Клиент сверяет локальные файлы с манифестом: если хеши совпадают, скачивание пропускается (Incremental Update).
        - Клиент проверяет MD5 скачанных данных перед записью на диск для защиты от поврежденных пакетов.
    - **[NEW] Dev Mode:** Добавлена опция `dev_mode: true` в конфиг клиента.
        - Позволяет игнорировать совпадение номеров версий.
        - Форсирует проверку изменений в файлах (MD5 Sync) при каждом запуске для ускорения разработки.

### Changed
- **Архитектура Клиента (RPi Optimization):**
    - **[PERFORMANCE]** Полностью удалена компиляция UI файлов (`.ui` -> `.py`) на стороне клиента.
    - Клиент теперь получает готовые скомпилированные интерфейсы (`resources/ui_done`) с сервера во время обновления.
    - Это снижает время запуска на Raspberry Pi и устраняет необходимость в dev-зависимостях (PySide6 Tools) на клиенте.
- **Точка входа (el_cliento.py):**
    - Добавлен вызов `cliento_updater.check_and_update()` перед инициализацией GUI.
    - Удалены импорты `manager_compile` и логика перекомпиляции.

- **Настройки (Bandito & Cliento):**
    - Ручное управление `IP Source` вместо авто-определения (теперь поле редактируемое и сохраняемое).
