# Changelog: Tune Plugin

## [0.0.0.9] 2026-03-09 - Полное управление звуком и громкостью
### Добавлено
- **Универсальный мониторинг:**
  - Внедрен класс `AudioStatusListener` для одновременного отслеживания статуса (Mute) и уровня громкости (Volume) для микрофонов и динамиков.
  - Поддержка отслеживания изменений громкости в реальном времени через системные настройки Windows.
- **Управление громкостью:**
  - Реализована двусторонняя синхронизация ползунков громкости `sound_volume_slider` и `mic_volume_slider` между клиентом и сервером.
  - Добавлена поддержка отправки значений громкости с клиента через "композитный ID" (`id|value`) для обхода ограничений протокола Core.
- **Управление звуком (Mute Sound):**
  - Реализовано включение/выключение звука текущего устройства через кнопку `sound_mute_toolB` (клиент) и `test_mute_sound_toolB` (сервер).
  - Автоматическая синхронизация состояния кнопки Mute при смене активного аудиоустройства.

### Изменено
- **Оптимизация и рефакторинг:**
  - Реализован универсальный метод `_set_volume_from_client` с использованием конфигурационного словаря, устраняющий дублирование кода для звука и микрофона.
  - Объединены разрозненные слушатели в единый поток `AudioStatusListener`.
  - Улучшен парсинг `btn_id` в `handle_button_press`: добавлена единая логика обработки слайдеров громкости.
  - Полная очистка кода от отладочных `print` сообщений для повышения производительности в продакшене.
  - Добавлена документация в `TuneClientoManager` по использованию композитных ID (`id|value`) для стабильной передачи данных.

## Мониторинг статуса микрофона в реальном времени

### Добавлено
- **Фоновый мониторинг микрофона:**
  - Реализован класс `MicStatusListener` (QThread) для отслеживания изменений статуса Mute/Unmute на системном уровне.
  - Автоматическое обновление UI (кнопка `test_mute_toolB`) при изменении статуса микрофона внешними средствами (горячие клавиши, кнопки на гарнитуре, системный микшер).
  - Динамическое переподключение слушателя при смене активного микрофона в комбобоксе `mic_01_comboB`.
- **Синхронизация состояния:**
  - Любое внешнее изменение статуса теперь автоматически сохраняется в конфиг и рассылается всем подключенным клиентам через Core Broadcast.

### Изменено
- **TnAudioManager:** Добавлены методы `start_mic_listening` и `stop_mic_listening` для управления жизненным циклом фонового потока.
- **Потокобезопасность:** Внедрена корректная инициализация COM-интерфейсов (`pythoncom.CoInitialize`) внутри рабочего потока для предотвращения конфликтов с основным циклом Qt.

## Удаленное управление и синхронизация мута микрофона

### Добавлено
- **Удаленное управление микрофоном:**
  - Кнопка `mic_mute_toolB` на клиенте теперь отправляет команду на сервер для переключения режима беззвучия.
  - Реализована полная синхронизация состояния кнопки (нажата/отжата) и иконки между всеми подключенными клиентами и сервером.
- **Индикация состояния:**
  - Текстовая индикация ("Mute"/"Muted") и автоматическая смена иконки микрофона в зависимости от режима.
  - Поддержка динамического свойства `isMuted` для расширенной стилизации через CSS.

### Исправлено
- **Критическая ошибка Pycaw:** Исправлено падение `[Tn] Mute Mic error: 'AudioDevice' object has no attribute 'Activate'` путем корректного обращения к внутреннему COM-объекту `_dev` (IMMDevice).
- **Инициализация при старте:** Исправлено отображение состояния кнопки мута при загрузке — теперь клиент сразу считывает актуальный флаг `input_devices_muted` из конфига и устанавливает `setChecked`.

### Изменено
- **Сохранение состояния:** Флаг `input_devices_muted` теперь персистентно сохраняется в `config_tune.json` при каждом переключении и восстанавливается при перезагрузке.
- **TnAudioManager:** Добавлены методы `set_mute_mic` и `is_mic_muted` для прямого управления системным микшером Windows через API IMMDevice.

## [0.0.0.8] 2026-03-09 - Синхронизация Bluetooth и микрофонов с защитой от оффлайна

### Добавлено
- **Поддержка Bluetooth:** Автоматическое обнаружение и фильтрация сопряженных BT-устройств (AirPods, Stereo и др.) в списке `bt_audiD_comboB`.
- **Поддержка микрофонов:** Отображение полного списка активных входных устройств в `mic_01_comboB`.
- **Защита от оффлайна:** 
  - На стороне клиента: элементы интерфейса (надписи/кнопки) для отключенных устройств автоматически становятся неактивными (`setEnabled(False)`).
  - На стороне сервера: проверка доступности устройства перед выполнением переключения системного аудиовыхода.
- **Улучшенная синхронизация:** Рассылка в реальном времени выбранных Bluetooth и Mic устройств на все подключенные клиенты.
- **Отслеживание активных устройств:** Сервер теперь включает список реально подключенных `active_devices` в каждое обновление конфига.

### Исправлено
- **Стабильность конфигурации:** Исправлена логика "жесткого списка" — пользовательские слоты Audio 1 и Audio 2 больше не перезаписываются полным списком активных устройств при старте.
- **Анимация UI:** Ограничено срабатывание анимации `eventFilter` только для кнопок (`QAbstractButton`), что предотвращает "клик" по статичным элементам, таким как `QGroupBox`.
- **Логика переключения:** Восстановлен отсутствующий метод `_select_device_by_index`, обеспечивающий надежное переключение звука с RPi.

### Изменено
- Обновлен `config_tune.json` для раздельного хранения настроек BT и микрофонов.
- Рефакторинг `TnAudioManager` для одновременной обработки потоков входных и выходных устройств.

## [0.0.0.6] - 2026-03-08

### Added
- **Интеграция с Core:**
  - Конструктор `TuneBanditoPlugin(plugin_path, core=None)` принимает опциональный `core` от ядра.
  - Рассылка клиентам выполняется через `core.com.broadcast("TUNE_CONFIG_UPDATE", self.config)` вместо прямого `requests.post`.
  - При отсутствии `core` выводится краткое предупреждение, broadcast не выполняется.

### Changed
- **Логи (оптимизация):**
  - Префикс логов сокращён до `[Tn]`.
  - В `tune_bandito.py`: убраны избыточные сообщения (Save button pressed, Client pressed, Selecting device); оставлены только краткие сообщения об ошибках и состоянии.
  - В `tn_audio_manager.py`: вместо нескольких строк при обновлении устройств вывода выводится одна строка `[Tn] Output devices count: N`. Остальные логи (Requesting..., Default device detected, перебор устройств) убраны.
  - При переключении системного устройства вывода (`set_default_output_device`): убрана строка «Switching...», при успехе выводится одна строка `[TuneBandito] Default output device set to: {device_name}`.

## [0.0.0.5] - 2026-03-08

### Added
- **Realtime-синхронизация конфига:**
  - Broadcast `config_tune.json` при загрузке плагина и при любом сохранении конфигурации.
  - Клиент обрабатывает `TUNE_CONFIG_UPDATE`, сохраняет конфиг на диск и эмитит `config_updated`.
- **Клиент — аудиоустройства:**
  - Заполнение `audiD_01_lineE` и `audiD_02_lineE` из `config_tune.json` (ключ `output_devices`).
  - Подсветка выбранного устройства через dynamic property `selectedDevice` и стиль `QLineEdit[selectedDevice="true"]` в `style_tune_cliento.json`.
- **Кнопки выбора устройства (Cliento → Bandito):**
  - `audiD_01_toolB` и `audiD_02_toolB` отправляют `PLUGIN_BUTTON_PRESS` на сервер.
  - Сервер определяет устройство по индексу (0/1), устанавливает его системным по умолчанию, сохраняет в конфиг и рассылает обновление клиентам.
