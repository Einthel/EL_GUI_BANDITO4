# Changelog — плагин Shortcut

Все изменения, связанные с плагином **Shortcut**, вынесенные из основного [changelog.md](changelog.md).

Формат: [Keep a Changelog](https://keepachangelog.com/en/1.0.0/), версии — [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

---

## [0.0.1.18] - 2026-03-08

### Added
- **Интеграция с Core:**
  - Конструктор `ShortcutBanditoPlugin(plugin_path, core=None)` и `ShortcutBanditoManager(plugin_path, core=None)` принимают опциональный `core` от ядра.
  - Синхронизация с клиентами выполняется через `core.com.broadcast("SHORTCUT_ICON_UPDATE"|"SHORTCUT_CONFIG_UPDATE", data)` вместо прямого POST на `/api/broadcast`.

### Changed
- **Очистка кода:** удалены функция `broadcast_to_server` и импорт `requests` из `sh_bandito_service.py`. При отсутствии `core` в `sync_with_clients` выводится предупреждение, broadcast не выполняется.
- **Логи:** в `handle_button_press` убран дублирующий вывод «Remote press received» — информация о нажатии уже есть в логе Core (`PLUGIN_BUTTON_PRESS → slot N 'btn_id'`).

## [0.0.1.17] - 2026-03-08

### Changed
- **Глубокий рефакторинг архитектуры (Server-side):**
  - Логика редактора разделена на три уровня: `shortcut_bandito.py` (UI), `sh_bandito_manager.py` (Бизнес-логика) и `sh_bandito_service.py` (Системные сервисы).
  - Внедрена централизованная система управления данными через `ShortcutBanditoManager`.
  - Все низкоуровневые операции (I/O, сетевое вещание, выполнение команд) вынесены в `sh_bandito_service.py`.
- **Оптимизация навигации по страницам:**
  - Реализована автоматическая переиндексация страниц при удалении для предотвращения разрывов в нумерации.
  - Исправлена блокировка кнопок навигации при изменении структуры страниц.
- **Улучшение стабильности:**
  - Исправлена ошибка загрузки плагина, связанная с некорректным обращением к `Qt.QSize` (заменено на `QSize` из `QtCore`).
  - Улучшена обработка входящих команд от ядра сервера: внедрена гибкая сигнатура метода `handle_button_press` и поддержка ID кнопок без указания страницы (fallback на `page_1`).
  - Исправлена поддержка структуры JSON-файла префабов с корневым ключом `"prefabs"`.

### Fixed
- Исправлен путь к файлу префабов в менеджере: `prefab_shortcut.json` → `prefab_but_shortcut.json`.
- Исправлено отсутствие визуального обновления навигации в UI при удалении страниц.

## [0.0.1.16] - 2026-03-02

### Added
- **Полная поддержка звуковой подсистемы (Server & Client):**
  - В `shortcut_bandito.py` (сервер) и `shortcut_cliento.py` (клиент) внедрена система `ElSoundManager`.
  - Реализована автоматическая привязка звуков к основным кнопкам сетки (1-12) через `el_sound_config.json`.
  - Добавлена озвучка кнопок управления страницами: `add_page`, `remove_page`, `next_page`, `back_page`.
  - Реализована трехуровневая логика воспроизведения:
    1. Поиск в глобальном конфиге (`plugins -> shortcut`).
    2. Поиск в свойствах кнопки (`sound_path` в `button_shortcut.json`).
    3. Fallback на `button_click.wav` из ресурсов плагина.
- **Интеграция с ядром:**
  - Плагин теперь корректно считывает состояние звука (включен/выключен) из соответствующих конфигов сервера (`el_bandito_config.json`) и клиента (`el_cliento_config.json`).

## [0.0.1.15] - 2026-03-02

### Added
- **Интеграция звуковой подсистемы (Sound System):**
  - В `shortcut_cliento.py` добавлена поддержка `ElSoundManager` для воспроизведения звуков при нажатии кнопок.
  - Реализована логика приоритетного воспроизведения: если в конфиге кнопки (`button_shortcut.json`) указан `sound_path`, проигрывается индивидуальный файл, иначе — стандартный `button_click.wav` из ресурсов плагина.
  - В `sh_cliento_manager.py` добавлен метод `get_button_props` для извлечения метаданных кнопки (включая пути к звукам) на стороне клиента.

## [0.0.1.13.23] - 2026-03-01

### Added
- **Архитектурное разделение логики (Refactoring):**
  - Логика плагина разделена на три модуля: `shortcut_cliento.py` (UI), `sh_cliento_manager.py` (Логика/Сокеты) и `sh_cliento_service.py` (I/O/Сервисы).
  - Введена система безопасного управления соединениями сигналов (`_click_connection`) для предотвращения утечек и ошибок при перерисовке интерфейса.
  - Поддержка гибких импортов для работы плагина как в составе пакета, так и при динамической загрузке.
- **Система анимации кнопок (EventFilter + QPropertyAnimation):**
  - Реализована плавная анимация "вдавливания" кнопок при нажатии (изменение `geometry` к центру).
  - Настройки анимации вынесены в `style_shortcut_cliento.json` (`duration`, `scale_factor`, `easing_curve`).
  - Поддержка различных кривых сглаживания (по умолчанию `OutQuad`).

### Changed
- **Оптимизация производительности и логирования:**
  - Глобальная очистка консоли от дебаг-шума: удалены избыточные `print` при загрузке конфигов, иконок и обмене данными с сервером.
  - Оптимизирован процесс обновления (Updater): детальные логи проверки MD5 и загрузки файлов переведены в режим `DEBUG`.
  - Итоговые отчеты об обновлении теперь выводятся одной строкой на плагин.
- **Исправление манифеста:**
  - Скорректированы пути в `shortcut_manifest.json` для соответствия новой структуре файлов.
  - Добавлены недостающие сервисные модули в карту синхронизации.
- **Оптимизация EventFilter:**
  - Централизованная обработка событий `MouseButtonPress`, `MouseButtonRelease` и `Leave` для всех кнопок плагина.
  - Предотвращение конфликтов анимаций при быстрых кликах через принудительную остановку предыдущих процессов.

### Fixed
- **Зависание состояния кнопок при навигации:**
  - Исправлена проблема "залипания" hover-эффекта на кнопках переключения страниц.
  - Реализован принудительный сброс фокуса и атрибута `WA_UnderMouse` для всех кнопок на старой и новой страницах при смене индекса в `StackedWidget`.

## [0.0.1.11] - 2026-02-28

### Added
- **Система стилей клиента (style_shortcut_cliento.json):**
  - Реализована загрузка внешних стилей из JSON-конфига для клиентской части плагина.
  - Поддержка селекторов ID (`#stream_cliento`, `#Button_stackedWidget`) и общих селекторов типов (`QToolButton`, `QFrame`).
  - Визуальное оформление кнопок: градиентный фон, скругление углов (15px), рамки 2px и эффект "проседания" при нажатии.
- **Процедурная генерация страниц (Client):**
  - Автоматическое назначение имен объектов (`objectName`) для всех динамически создаваемых виджетов (фреймов, лейаутов, кнопок) для корректной работы CSS-селекторов.
  - Принудительная очистка текста на кнопках (`ToolButtonIconOnly`) для всех страниц, включая процедурные.

### Changed
- **Логика отображения кнопок:**
  - Полный переход на иконочный интерфейс: подписи к кнопкам (Title) теперь используются только в редакторе и конфиге, на самом виджете текст принудительно скрыт.
  - Масштабирование иконок: установлен фиксированный размер 150x150 для всех кнопок клиента, игнорируя старые значения из конфига (70px).
  - Оптимизация загрузки стилей: метод `apply_plugin_styles` в `shortcut_cliento.py` теперь полностью соответствует логике редактора.

## [0.0.1.9] - 2026-02-10 (Shortcut & Core)

### Changed
- **Рефакторинг версий плагина Shortcut:**
  - Устранён конфликт имён файлов версий: `plugins/shortcut/ver` → `ver_shortcut`.
  - Обновлены манифесты и логика апдейтеров для поддержки новых имён.

### Added
- **Система префабов (Shortcut):**
  - Реализована библиотека шаблонов кнопок (префабов), хранящаяся в `prefab_but_shortcut.json`.
  - **UI дерева:** `TreeWidget` отображает список доступных префабов, сгруппированных по категориям (Application, Shortcut, Action, Other).
  - **Сохранение:** При создании новой кнопки открывается диалог `SavePrefabDialog` для сохранения кнопки как именованного шаблона.
  - **Валидация:** Проверка уникальности имени префаба с диалогом подтверждения перезаписи (`QuestionsDialog`).
  - **Управление:** Контекстное меню в дереве для удаления и переименования префабов.
- **Push-обновление контента (Live Update) для Shortcut:**
  - Мгновенная отправка новых иконок на клиент (`SHORTCUT_ICON_UPDATE`) при сохранении.
  - Мгновенная отправка обновлённого конфига (`SHORTCUT_CONFIG_UPDATE`).
  - Клиент принимает и сохраняет изменения без перезагрузки приложения.
