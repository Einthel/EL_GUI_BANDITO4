## [0.0.0.9] - 2026-02-24

### Added
- **Sound Plugin (Оптимизация I/O и UI):**
  - **Кэширование конфигурации:** В `sound_save_load.py` внедрена система кэширования с проверкой `mtime` файла. Чтение с диска происходит только при реальном изменении файла, что устраняет фризы при переключении плагинов.
  - **Умная запись:** Метод `save_sound_config` теперь сравнивает новые данные с кэшем и выполняет запись на диск только при наличии фактических изменений.
  - **Рефакторинг форм:** Из интерфейса удалены неиспользуемые элементы `audiD_03_comboB`, `audiD_03_lineE` и `audiD_03_toolB`. Код плагинов `sound_bandito.py` и `sound_cliento.py` очищен от соответствующих упоминаний.
  - **Атомарные обновления:** Переход на передачу только измененных полей в `save_sound_config` (например, только `mute_state` или `volume`), что снижает нагрузку на систему сохранения.
  - **Стабильность:** Исправлена критическая синтаксическая ошибка в `sound_save_load.py` (некорректная вложенность `try/except`), приводившая к падению приложения при сохранении.

## [0.0.0.8] - 2026-02-24

### Added
- **Sound Plugin (Стабилизация и логирование):**
  - **Дебаг-логирование:** Добавлен детальный вывод входящих команд от клиента (`SOUND_SET_VOLUME`, `SOUND_TOGGLE_MUTE`) в консоль сервера для отслеживания жизненного цикла запросов.
  - **Обработка ошибок COM:** Внедрены блоки `try/except` вокруг критических вызовов `pycaw` в `SoundCore`, что предотвращает аварийное завершение `el_bandito.py` при сбоях в подсистеме аудио Windows.
  - **Оптимизация Cooldown:** Значение кулдауна между системными вызовами в `SoundCore` скорректировано до 0.1с для обеспечения баланса между отзывчивостью интерфейса и стабильностью COM-соединения.
  - **Информативность:** Добавлены префиксы `[SoundBandito]` и `[SoundCore]` ко всем консольным сообщениям для облегчения фильтрации логов.
- **Sound Plugin (Синхронизация и защита от циклов):**
  - **Защита от "эха":** Внедрен механизм блокировки обратной трансляции (`_block_broadcast`). Теперь данные, пришедшие от сервера, сохраняются локально без повторной отправки на сервер, и наоборот.
  - **Инициализация:** Добавлен метод `apply_config_volume` на сервере для принудительной установки системной громкости из `config_sound.json` при старте.
  - **Синхронизация UI:** Реализовано автоматическое выставление значений слайдеров `sound_volume_slider` и `mic_volume_slider` при загрузке клиента на основе данных из конфига.
  - **Управление звуком:** Переход на использование `SetMasterVolumeLevelScalar` (pycaw) для точной установки системной громкости в диапазоне 0.0-1.0.
  - **Стабильность:** Добавлена проверка `hasattr(self.socket_client, 'send_command')` в `SoundCore`, что предотвращает падения при некорректной инициализации сокета строковым значением.
  - **Отладка:** Добавлен временный вывод в лог текущего значения громкости при перемещении слайдера на клиенте.

## [0.0.0.7] - 2026-02-24

### Added
- **Sound Plugin (Расширенная синхронизация громкости):**
  - Реализовано получение актуального уровня громкости системы (0-100%) через `GetMasterVolumeLevelScalar`.
  - Добавлена поддержка отслеживания изменений громкости в реальном времени через системные колбэки `OnNotify`.
  - Внедрен механизм **Debounce** (отложенное сохранение, 500мс) для записи уровня громкости в `config_sound.json`, что предотвращает избыточные операции ввода-вывода при быстром изменении звука.
  - **Архитектура:** Реализована потокобезопасная передача данных из COM-потока в основной поток Qt через промежуточный объект-эмиттер (`AudioSignalEmitter`) и сигналы. Это устранило возможные зависания и ошибки доступа к памяти.
  - Добавлена принудительная синхронизация громкости при инициализации плагина (считывание текущих системных значений и обновление конфига).
  - Оптимизировано логирование: добавлено информативное сообщение `Config successfully saved and send to cliento` при завершении цикла сохранения.

## [0.0.0.6] - 2026-02-23

### Added
- **Sound Plugin (Системная синхронизация):**
  - Реализовано автоматическое отслеживание изменений звука в Windows через `IAudioEndpointVolumeCallback`.

  - Добавлена обработка команды `SOUND_TOGGLE_MUTE` на сервере для физического переключения звука в ОС.
  - Оптимизирована логика `SoundCore`: методы `set_mute` и `toggle_mute` теперь разделяют локальное обновление UI (мгновенный отклик) и системное выполнение на сервере.
  - **Оптимизация:** Проведена полная чистка кода от отладочных логов (`print`) и неиспользуемого функционала (`sound_up`/`down`).
